## ✨ Key Features

### 🤖 AI-Powered Assistant (Star Feature)
- **Gemini 1.5 Flash** chatbot with full Urdu/English support
- **PDF → Quiz Generator** — Upload any PDF and auto-generate MCQ / True-False questions (Easy / Medium / Hard)
- **Upload & Summarize** — Drag & drop PDF/DOC/TXT → get a summary or cheat sheet instantly
- Voice input, file attachment context, suggestion chips, typing animation
- Chat, quiz, and summary history saved locally

### 🎓 Student Panel
- Dashboard with real-time stats, progress bars, quiz countdowns
- Course browsing with category & search filters, enrollment & lesson tracking
- Assignment submission (live link + screenshot upload via Cloudinary) with drag & drop
- Hackathons & Competitions — register, track deadlines, submit projects
- Programs (bootcamps, workshops) — join/leave, seat availability
- Profile management with Cloudinary picture upload

### 👩‍🏫 Teacher Command Center
- Live activity ticker strip + KPI cards (students, revenue, completions)
- Course builder (create / edit / publish / draft)
- Assignment reviewer — approve or reject with written feedback
- Quiz builder — MCQ with 4 options, schedule date/time/duration
- Student roster with per-student enrollment & assignment stats
- Analytics: monthly enrollments, revenue, completion rates (Recharts)

### 🏠 Landing Page
- MP4 video hero background with opacity overlay
- GSAP scroll animations + Lenis smooth scrolling
- Background color transform on scroll (black → green tint)
- Animated ticker strip, pricing plans (PKR), testimonials, CTA

### 🔐 Auth & Security
- JWT-based auth with role selection (Student / Teacher)
- Bcrypt password hashing, role-protected routes
- Helmet headers, CORS whitelist, Rate limiting (200 req / 15 min)

## ⚙️ Local Setup

### Prerequisites
- Node.js v18+
- MongoDB Atlas account
- Cloudinary account
- Stripe account
- Google Gemini API key

### 1. Clone the repository

```bash
git clone https://github.com/your-username/una-lms.git
cd una-lms
```

### 2. Backend setup

```bash
cd server
npm install
```

Create `server/.env`:

```env
PORT=5000
MONGODB_URI=your_mongodb_atlas_uri
JWT_SECRET=your_jwt_secret
JWT_EXPIRES_IN=7d

CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_WEBHOOK_SECRET=your_webhook_secret

CLIENT_URL=http://localhost:5173
```

```bash
npm run dev   # starts on http://localhost:5000
```

### 3. Frontend setup

```bash
cd ../client
npm install
```

Create `client/.env`:

```env
VITE_API_URL=http://localhost:5000
VITE_GEMINI_API_KEY=your_gemini_api_key
VITE_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
```

```bash
npm run dev   # starts on http://localhost:5173
```

## 🌐 Deployment

### Frontend → Netlify

The `client/` folder contains a `netlify.toml` for automatic SPA routing.

```bash
cd client
npm run build
# Deploy the dist/ folder to Netlify
# Or connect your GitHub repo directly
```

### Backend → Vercel

The `server/vercel.json` is already configured.

```bash
cd server
vercel --prod
# Add all environment variables in Vercel dashboard
```

## 🔌 API Routes

| Method | Route | Auth | Description |
|---|---|---|---|
| `POST` | `/api/auth/signup` | ❌ | Register (student/teacher) |
| `POST` | `/api/auth/login` | ❌ | Login, get JWT |
| `GET` | `/api/student/*` | ✅ Student | Student panel data |
| `GET/POST` | `/api/teacher/*` | ✅ Teacher | Teacher panel data |
| `POST` | `/api/payment/checkout` | ✅ | Stripe checkout |
| `POST` | `/api/payment/webhook` | — | Stripe webhook |
| `GET` | `/api/health` | ❌ | Health check |
